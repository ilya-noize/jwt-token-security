# jwt-token-security
### Основные компоненты конфигурации Spring Security

| Компонент             | Назначение                            |
|-----------------------|---------------------------------------|
| SecurityFilterChain   | Главная цепочка фильтров безопасности |
| AuthenticationManager | Управляет процессом аутентификации    |
| UserDetailsService    | Загружает данные пользователя         |
| PasswordEncoder       | Шифрует и проверяет пароли            |
| SecurityContext       | Хранит информацию об аутентификации   |

### Практические советы


1. Всегда используйте PasswordEncoder для хеширования паролей
2. Отключайте CSRF для REST API
3. Устанавливайте STATELESS сессии при использовании JWT
4. Настройте CORS для кросс-доменных запросов
5. Используйте роли и authorities для гибкой авторизации


### Когда использовать сессии vs JWT

#### Используйте сессии когда:

- Монолитное приложение
- Нужна простая инвалидация логаута
- Работаете в основном с браузерами

#### Используйте JWT когда:

* Микросервисная архитектура
* Мобильные приложения
* Stateless масштабирование
* SSO (Single Sign-On)
